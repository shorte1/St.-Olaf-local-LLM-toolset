Prompt: Explain the difference between TCP and UDP protocols and when you would use each.
========================================
TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are two fundamental protocols used for transmitting data over the internet. While both protocols are used for packet switching, they have distinct differences in terms of their approach to error handling, reliability, and speed.

**TCP:**

1. **Connection-oriented**: TCP establishes a connection between the sender and receiver before sending data. This allows for reliable, error-checked delivery of data.
2. **Reliable**: TCP ensures that data is delivered accurately and in the correct order.
3. **Error-checking**: TCP uses checksums to detect errors during transmission.
4. **Flow control**: TCP regulates the amount of data sent by both parties to prevent network congestion.
5. **Ordered delivery**: TCP guarantees that data packets will be received in the same order they were sent.

**UDP:**

1. **Connectionless**: UDP does not establish a connection before sending data, making it faster and more suitable for real-time applications.
2. **Unreliable**: UDP does not guarantee the delivery or accuracy of data; errors may occur during transmission.
3. **No error-checking**: UDP relies on the receiving party to detect errors.
4. **No flow control**: UDP allows for full-speed transmission, which can lead to network congestion if not managed properly.
5. **Unordered delivery**: UDP does not guarantee that data packets will be received in the same order they were sent.

**When to use each protocol:**

1. **Use TCP for:**
 * Applications requiring reliable, accurate, and ordered data transfer, such as:
 + File transfers
 + Email
 + Web browsing
2. **Use UDP for:**
 * Real-time applications that can tolerate some packet loss or errors, such as:
 + Online gaming
 + Video streaming
 + VoIP (Voice over Internet Protocol)
 + DNS lookups

In summary:

* TCP is a connection-oriented protocol that provides reliable and error-checked data transfer.
* UDP is a connectionless protocol that prioritizes speed and efficiency over reliability.

Choose the right protocol based on your application's requirements, trade-offs between performance and reliability.
