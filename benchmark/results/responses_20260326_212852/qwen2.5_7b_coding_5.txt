Prompt: Write a REST API endpoint in Python using Flask that accepts a JSON body with a username and password, validates that neither is empty, and returns a JWT token.
========================================
Sure! Below is an example of how you can create a simple REST API endpoint using Flask that accepts a JSON body containing `username` and `password`, validates if they are not empty, and then returns a JWT (JSON Web Token) if the validation passes.

First, make sure to install the necessary packages:
```bash
pip install flask flask-jwt-extended python-jose[cryptography]
```

Now, let's write the Python code for this API:

```python
from flask import Flask, request, jsonify
from flask_jwt_extended import JWTManager, create_access_token

app = Flask(__name__)

# Configure the secret key and algorithm for the JWT
app.config['JWT_SECRET_KEY'] = 'your-secret-key'  # Change this!
jwt = JWTManager(app)

@app.route('/login', methods=['POST'])
def login():
    if not request.is_json:
        return jsonify({"msg": "Missing JSON in request"}), 400

    username = request.json.get('username', None)
    password = request.json.get('password', None)

    # Validation checks
    if not username or not password:
        return jsonify({"msg": "Username and password are required"}), 400

    # You can add more complex logic here for authentication, e.g., querying a database.
    # For this example, let's assume we always return a successful login response.

    access_token = create_access_token(identity=username)
    return jsonify(access_token=access_token)

if __name__ == '__main__':
    app.run(debug=True)
```

### Explanation:
1. **Dependencies**:
   - `flask`: The main framework for creating the API.
   - `flask-jwt-extended`: A library to manage JWT tokens easily with Flask.

2. **Configuration**:
   - `JWT_SECRET_KEY`: A secret key used for signing the JWT token. Make sure to keep this secure and change it from a placeholder value.
   - The `JWTManager` is initialized with this secret key.

3. **Endpoint Definition (`/login`)**:
   - The endpoint `/login` accepts POST requests.
   - It expects JSON data containing `username` and `password`.
   - If the request does not contain JSON, it returns a 400 Bad Request error.
   - If `username` or `password` is missing in the JSON body, it returns an appropriate error message.

4. **Token Generation**:
   - A valid username (for simplicity, any non-empty string will do) is used to generate a JWT token using `create_access_token`.
   - The generated token is returned as a response.

5. **Running the Application**:
   - The application runs on the local server with debugging enabled (`app.run(debug=True)`).
