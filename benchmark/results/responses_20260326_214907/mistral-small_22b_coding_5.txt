========================================
 To create a REST API endpoint in Python using Flask that accepts a JSON body with a username and password, validates them, and returns a JWT token, you'll need to follow these steps:

1. Install the required packages if you haven't already:
    ```bash
    pip install flask pyjwt
    ```

2. Create a Flask application with an endpoint for authentication and JWT generation.

Here is a complete example:

```python
from flask import Flask, request, jsonify
import jwt
import datetime

app = Flask(__name__)

# Secret key for signing the JWT. In production, use a secure random string.
SECRET_KEY = 'your_secret_key'

@app.route('/login', methods=['POST'])
def login():
    # Get JSON data from request
    data = request.get_json()

    # Validate that username and password are provided
    if not data or 'username' not in data or 'password' not in data:
        return jsonify({"message": "Missing username or password"}), 400

    # Extract username and password
    username = data['username']
    password = data['password']

    # TODO: Add authentication logic here (e.g., check against a database)

    # For demonstration purposes, let's assume the credentials are valid
    if username == 'user' and password == 'pass':  # Replace with actual validation logic
        # Generate the JWT token
        token = jwt.encode({
            'username': username,
            'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=30)  # Token expiration time (e.g., 30 minutes)
        }, SECRET_KEY, algorithm='HS256')

        return jsonify({"token": token}), 200
    else:
        return jsonify({"message": "Invalid username or password"}), 401

if __name__ == '__main__':
    app.run(debug=True)
```

### Explanation:
1. **Importing Libraries**:
   - `Flask` and `request` are used to create the Flask application and handle incoming requests.
   - `jsonify` is used to return JSON responses from the server.
   - `jwt` is used to encode JWT tokens.
   - `datetime` is used to set an expiration time for the token.

2. **Configuring Flask**:
   - An instance of Flask is created and configured with a secret key (which should be secure in production).

3. **Defining the Login Endpoint**:
   - The `/login` endpoint accepts POST requests.
   - It retrieves JSON data from the request body.
   - It validates that both `username` and `password` are provided.
   - A placeholder authentication logic checks if the credentials match predefined values (this should be replaced with actual database or user management system validation).
   - If the credentials are valid, it generates a JWT token with an expiration time of 30 minutes and returns it in the response.
   - If the credentials are invalid, it returns an error message.

4. **Running the Application**:
   - The application runs in debug mode for development purposes.
