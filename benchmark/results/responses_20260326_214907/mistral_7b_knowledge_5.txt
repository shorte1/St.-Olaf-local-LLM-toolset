Prompt: Explain the difference between TCP and UDP protocols and when you would use each.
========================================
 TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are two fundamental network protocols used for data communication over the internet. The main differences between them lie in their reliability, order of delivery, and efficiency.

1. Reliability:
TCP is a connection-oriented, reliable, and stream-oriented protocol. It ensures that all the data packets sent from one endpoint to another reach their destination without errors. TCP does this by using mechanisms like flow control, congestion avoidance, retransmission of lost or corrupted packets, and acknowledgement of received data packets.

On the other hand, UDP is a connectionless, unreliable protocol. It doesn't guarantee that data packets will be delivered to the destination or that they will arrive in the same order they were sent. Since UDP does not retransmit lost or corrupted packets, it can lead to data loss but offers faster transmission speeds due to its simplicity.

2. Order of Delivery:
TCP ensures the correct order of delivery for data packets by keeping track of packet sequence numbers. When packets arrive out of order, TCP will rearrange them before forwarding them to the application layer. UDP does not guarantee the order of delivery, and it's up to the application layer to deal with out-of-order packets if necessary.

3. Efficiency:
TCP is more efficient in bandwidth-intensive applications where data reliability is crucial, such as web browsing, file transfers, email services, and VoIP calls using codecs like Opus or G.711. However, TCP's handshaking process and packet retransmission can introduce extra overhead, which may impact performance in latency-sensitive applications.

UDP is more suitable for real-time applications that require low latency, such as online gaming, streaming services like YouTube Live, and live sports broadcasting. In these cases, a slight loss of packets (up to a certain limit) can be tolerated since the application layer can handle missing or corrupted data without causing significant issues.

4. When to use each protocol:
- Use TCP for applications that require reliable data transfer, such as web browsing, file transfers, and VoIP calls using codecs like Opus or G.711.
- Use UDP for real-time applications with low latency requirements, including online gaming, streaming services like YouTube Live, and live sports broadcasting.
