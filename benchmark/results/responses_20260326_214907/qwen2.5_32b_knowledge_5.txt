Prompt: Explain the difference between TCP and UDP protocols and when you would use each.
========================================
TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are both fundamental protocols in the Internet protocol suite used for data transmission over networks, but they serve different purposes and have distinct characteristics.

### TCP (Transmission Control Protocol)
- **Connection-Oriented**: Before any data transfer occurs, a connection needs to be established between the sender and receiver. This is often referred to as the three-way handshake.
- **Reliable**: Ensures that all packets are received in order by retransmitting lost or damaged packets. It also checks for errors using checksums and provides mechanisms for flow control and congestion control.
- **Ordered Delivery**: TCP guarantees that segments will be delivered in the exact order they were sent, even if some of them arrive out of sequence.
- **Slower**: Due to its reliability features (retransmission, sequencing), it tends to have higher overhead and can result in slower data transfer compared to UDP.

### Use Cases for TCP
- When reliability is crucial: applications that require all data to be delivered correctly and completely such as web browsing (HTTP/HTTPS), email (SMTP), file transfers (FTP), and remote terminal access (SSH).

### UDP (User Datagram Protocol)
- **Connectionless**: There’s no formal connection setup. Packets are sent individually, and delivery is not guaranteed.
- **Unreliable**: Does not guarantee that packets will arrive at all or in the same order they were sent. No retransmission of lost data happens automatically.
- **Faster**: Because UDP has fewer protocol overheads compared to TCP (no connection setup/teardown, no sequencing control), it can be faster and more suitable for real-time applications where speed is preferred over reliability.

### Use Cases for UDP
- Real-time multimedia streaming: Used in VoIP (Voice over IP) communications, video conferencing, online gaming, and live broadcasts where some packet loss is acceptable.
- DNS queries: Often uses UDP because it’s faster to send a request and get an answer without the overhead of establishing a connection.

In summary, TCP is chosen when reliability and ordered delivery are essential, while UDP is preferred for applications that prioritize speed over guaranteed delivery. The choice between TCP and UDP should be based on the specific requirements of the application or service being developed.
