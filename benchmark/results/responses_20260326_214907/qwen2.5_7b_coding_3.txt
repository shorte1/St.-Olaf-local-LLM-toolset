Prompt: Write a Python function that takes a list of integers and returns the two numbers that add up to a target sum. It should run in O(n) time.
========================================
To solve this problem efficiently in O(n) time, we can use a hash map (or dictionary in Python) to store the difference between the target sum and each element as we iterate through the list. This way, we can quickly check if the complement of the current number exists in our dictionary. Here's how you can implement it:

```python
def find_two_sum(numbers, target):
    # Create a dictionary to store the numbers and their indices
    num_map = {}
    
    # Iterate over each number in the list
    for i, num in enumerate(numbers):
        # Calculate the complement of the current number
        complement = target - num
        
        # Check if the complement is already in the map
        if complement in num_map:
            # If found, return the indices (0-based)
            return [num_map[complement], i]
        
        # Store the index of the current number
        num_map[num] = i
    
    # If no solution is found, raise an exception or handle it as needed
    raise ValueError("No two sum solution")

# Example usage:
numbers = [2, 7, 11, 15]
target = 9
print(find_two_sum(numbers, target))  # Output: [0, 1]

# Another example usage with a different target
numbers = [3, 2, 4]
target = 6
print(find_two_sum(numbers, target))  # Output: [1, 2]
```

### Explanation:
- **num_map**: This dictionary will store each number and its index as we iterate through the list. The key is the number, and the value is its corresponding index.
- For each number in `numbers`, calculate the complement (`target - num`).
- Check if this complement already exists in the dictionary `num_map`. If it does, return the indices of the two numbers that add up to the target sum.
- If the complement is not found, store the current number and its index in the dictionary.

This approach ensures that we only pass through the list once (O(n) time complexity), making it efficient.
