Prompt: Write a REST API endpoint in Python using Flask that accepts a JSON body with a username and password, validates that neither is empty, and returns a JWT token.
========================================
Sure! Below is an example of how you can create a simple REST API using the Flask framework in Python to accept a JSON body containing `username` and `password`, validate them, and return a JSON Web Token (JWT) if they are valid.

First, ensure you have Flask and PyJWT installed. You can install these packages via pip:

```bash
pip install flask pyjwt
```

Now, let's write the code for your API endpoint:

```python
from flask import Flask, request, jsonify
import jwt
import datetime

app = Flask(__name__)

# Secret key to sign JWTs (should be kept secret in a production environment)
SECRET_KEY = 'your_secret_key'

@app.route('/login', methods=['POST'])
def login():
    try:
        # Parse the JSON body from the request
        data = request.get_json()
        
        if not data or 'username' not in data or 'password' not in data:
            return jsonify({"error": "Invalid input, username and password are required."}), 400
        
        username = data['username']
        password = data['password']
        
        # Here we would normally check against a database
        if username == '' or password == '':
            return jsonify({"error": "Username or password cannot be empty."}), 400

        # Generate JWT token with an expiration time (e.g., 1 hour)
        token = jwt.encode(
            {
                'username': username,
                'exp': datetime.datetime.utcnow() + datetime.timedelta(hours=1)
            },
            SECRET_KEY,
            algorithm='HS256'
        )

        return jsonify({"token": token}), 200

    except Exception as e:
        return jsonify({"error": str(e)}), 400

if __name__ == '__main__':
    app.run(debug=True)
```

### Explanation:

1. **Imports and Flask Setup**:
   - Import necessary modules: `Flask`, `request`, `jsonify`.
   - Define a secret key for signing JWTs (keep this secure in production).
   
2. **Endpoint Definition (`/login`)**:
   - The endpoint accepts POST requests.
   - The JSON body is expected to contain keys `username` and `password`.
   - Validate that both `username` and `password` are present and non-empty.
   - If the validation passes, generate a JWT token with an expiration time of 1 hour using PyJWT.

3. **Running the App**:
   - The Flask app is run in debug mode for easier development.

### Running the Code:

To run this script, save it to a file (e.g., `app.py`), and then use the following command to start your Flask application:

```bash
python app.py
```

This will start the Flask server. You can test the endpoint using tools like `curl`, Postman, or any other HTTP client by sending a POST request with a JSON body containing `username` and `password`.

Example cURL Command:
```bash
curl -X POST http://127.0.0.1:5000/login -H "Content-Type: application/json" -d '{"username":"testuser", "password":"testpass"}'
```

This will return a JSON response containing the JWT token if the request is valid.
