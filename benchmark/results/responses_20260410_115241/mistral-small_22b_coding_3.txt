Prompt: Write a Python function that takes a list of integers and returns the two numbers that add up to a target sum. It should run in O(n) time.
========================================
 To achieve this, we can use a hash map (dictionary in Python). The idea is to iterate through the list once while keeping track of the complements we have seen so far. For each number, we check if its complement exists in the dictionary. If it does, we return those two numbers as they add up to the target sum.

Here's a Python function that implements this approach:

```python
def find_two_sum(nums, target):
    # Create a hash map (dictionary) to store the complement and its index
    num_dict = {}

    # Iterate through the list of numbers
    for i, num in enumerate(nums):
        # Calculate the complement
        complement = target - num

        # Check if the complement is already in the dictionary
        if complement in num_dict:
            return (complement, num)  # Return the two numbers that add up to the target sum

        # Store the number and its index in the hash map
        num_dict[num] = i

    # If no such pair is found, return None or raise an error
    return None

# Example usage:
nums = [2, 7, 11, 15]
target = 9
print(find_two_sum(nums, target))  # Output should be (2, 7) or (7, 2)
```

### Explanation:
1. **Initialization**: We initialize an empty dictionary `num_dict`.
2. **Iteration**: For each number in the list, we calculate its complement (i.e., `target - num`).
3. **Check Dictionary**: We check if this complement already exists in our dictionary. If it does, we have found two numbers that add up to the target sum and return them.
4. **Update Dictionary**: If the complement is not found, we store the current number and its index in the dictionary. This allows us to keep track of the numbers we have seen so far and their positions.
5. **Return Result**: If no such pair is found by the end of the iteration, we return `None`.

This approach ensures that we only iterate through the list once, giving us an O(n) time complexity.
