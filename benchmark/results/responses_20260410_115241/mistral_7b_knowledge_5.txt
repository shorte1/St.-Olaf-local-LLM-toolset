Prompt: Explain the difference between TCP and UDP protocols and when you would use each.
========================================
TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are two fundamental networking protocols used for communication over the internet, each with its unique characteristics, advantages, and disadvantages. Understanding their differences helps in selecting the right one for specific scenarios:

1. TCP (Transmission Control Protocol):
- Connection-oriented: TCP establishes a reliable connection between sender and receiver before transmitting data. It maintains this connection throughout the entire session to ensure that packets are delivered correctly, making it more suitable for applications where data integrity is crucial, such as web browsing or email transmission.
- Flow control and congestion control: TCP uses techniques like sliding window protocol and slow-start to manage the flow of data and prevent network congestion.
- Reliability: TCP guarantees that packets are delivered in the correct order without errors by using acknowledgments, retransmission, and sequence numbers.

2. UDP (User Datagram Protocol):
- Connectionless: Unlike TCP, UDP does not establish a connection before sending data. It simply sends datagrams, or packets, to their destination with minimal overhead, making it faster and more efficient for applications that require low latency but can tolerate some packet loss, such as video streaming or online gaming.
- No flow control or congestion control: UDP does not provide mechanisms like TCP's sliding window protocol or slow-start to manage data flow and prevent network congestion, resulting in a higher likelihood of dropped packets when the network is congested.
- Lack of reliability: UDP does not guarantee packet delivery, order, or error correction, relying on application-level mechanisms to handle these issues if necessary (e.g., retransmission).

In summary, you would use TCP for applications where data integrity and reliability are essential, such as web browsing, file transfer, and email transmission. Meanwhile, UDP is more suitable for low-latency real-time applications like streaming audio/video or online gaming, where some loss of packets can be tolerated without significantly impacting user experience.
