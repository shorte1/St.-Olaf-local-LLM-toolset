The provided Python code for calculating Fibonacci numbers has a common issue where it uses recursion without optimization. Specifically, the function recalculates many of the same Fibonacci numbers multiple times, leading to an inefficient and potentially very slow performance, especially for large values like `n=100`. This results in a time complexity of O(2^n) due to exponential growth.

Here's the code you provided:

```python
def fibonacci(n):
    if n == 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(100))
```

### Issues and Debugging

1. **Exponential Time Complexity**: The recursive approach recalculates the same Fibonacci numbers multiple times, leading to an inefficient algorithm.
2. **Potential Stack Overflow**: For very large `n` (like 100), this can cause a stack overflow error due to excessive recursion.

### Optimized Solution: Dynamic Programming

To optimize the solution, you can use dynamic programming techniques like memoization or an iterative approach with storage for previous results. Here is an optimized version using memoization:

```python
def fibonacci(n):
    if n == 0:
        return 0
    elif n == 1:
        return 1
    
    # Initialize a list to store Fibonacci numbers up to n
    fib = [0, 1] + [None] * (n - 1)
    
    # Fill the list using bottom-up approach
    for i in range(2, n + 1):
        fib[i] = fib[i-1] + fib[i-2]
    
    return fib[n]

print(fibonacci(100))
```

### Explanation

1. **Memoization**: We create a list `fib` to store the Fibonacci numbers up to `n`. This way, we avoid recalculating values that have already been computed.
2. **Bottom-Up Approach**: The loop starts from 2 and goes up to `n`, filling in each value based on previously computed values.

### Iterative Solution

Another approach is to use an iterative method without using recursion:

```python
def fibonacci(n):
    if n == 0:
        return 0
    elif n == 1:
        return 1
    
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    
    return b

print(fibonacci(100))
```

### Explanation

- **Iterative Calculation**: We start with the base cases `a` and `b`. Then, we iterate from 2 to `n`, updating the values of `a` and `b` in each iteration.

Both optimized methods will significantly reduce the time complexity to O(n), making the computation much faster for large values of `n`.

### Conclusion

By using dynamic programming or an iterative approach, you can efficiently compute Fibonacci numbers without running into performance issues or stack overflow errors.
