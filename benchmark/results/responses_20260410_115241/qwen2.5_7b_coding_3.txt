Prompt: Write a Python function that takes a list of integers and returns the two numbers that add up to a target sum. It should run in O(n) time.
========================================
To solve this problem efficiently with an `O(n)` runtime, you can use a hash map (or dictionary) to store the numbers as you iterate through the list. This allows for constant-time lookups, ensuring that the solution is efficient.

Here's a Python function that implements this approach:

```python
def two_sum(nums, target):
    # Create an empty dictionary to store the difference between the target and the current number.
    num_map = {}
    
    # Iterate through each number in the list.
    for i, num in enumerate(nums):
        # Calculate the complement of the current number with respect to the target.
        complement = target - num
        
        # Check if the complement is already in the dictionary.
        if complement in num_map:
            # If it is, we have found a pair that adds up to the target.
            return [num_map[complement], i]
        
        # Otherwise, store the current number with its index in the dictionary.
        num_map[num] = i
    
    # If no solution is found (though this should not happen given O(n) constraints), return an empty list.
    return []

# Example usage:
nums = [2, 7, 11, 15]
target = 9
print(two_sum(nums, target))  # Output: [0, 1] because nums[0] + nums[1] == 9

```

### Explanation:

- **num_map**: This dictionary stores each number and its index as we iterate through the list. The key is the number itself, and the value is its index.
  
- For each number `num` in the list:
  - We calculate the `complement`, which is `target - num`.
  - If this `complement` already exists in `num_map`, it means we have found a pair of numbers that add up to the target. The indices of these two numbers are returned.

- If no such pair is found during the iteration, an empty list is returned (though typically, given the constraints and input conditions, this should not happen).

This approach ensures that the function runs in `O(n)` time complexity because each lookup and insertion into the dictionary takes `O(1)` average-time.
