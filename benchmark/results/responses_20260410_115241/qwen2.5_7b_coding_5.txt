Prompt: Write a REST API endpoint in Python using Flask that accepts a JSON body with a username and password, validates that neither is empty, and returns a JWT token.
========================================
Certainly! Below is an example of how you can create a REST API endpoint in Python using the Flask framework to accept a JSON body containing `username` and `password`, validate that both fields are not empty, and return a JWT token.

First, ensure you have the necessary libraries installed:

```sh
pip install flask pyjwt
```

Here's an example implementation:

```python
from flask import Flask, request, jsonify
import jwt
import datetime

app = Flask(__name__)

# Secret key for signing the JWT token (keep this secure)
SECRET_KEY = 'your_secret_key'

def generate_token(username):
    payload = {
        "exp": datetime.datetime.utcnow() + datetime.timedelta(days=0, minutes=30),
        "iat": datetime.datetime.utcnow(),
        "sub": username
    }
    return jwt.encode(payload, SECRET_KEY, algorithm='HS256')

@app.route('/login', methods=['POST'])
def login():
    data = request.get_json()
    
    # Validate that both username and password are present
    if 'username' not in data or 'password' not in data:
        return jsonify({"error": "Username and password are required"}), 400

    username = data['username']
    password = data['password']

    # For the sake of this example, we will assume that any non-empty value for `username` is valid
    if not username or not password:
        return jsonify({"error": "Username and password are required"}), 400

    token = generate_token(username)

    return jsonify({
        "token": token.decode('utf-8')
    }), 200

if __name__ == '__main__':
    app.run(debug=True)
```

### Explanation:

1. **Flask Setup**: We import the necessary modules and create a Flask application.
2. **JWT Configuration**: The `generate_token` function creates a JWT token with an expiration time of 30 minutes from now.
3. **Endpoint Definition**: 
   - The `/login` endpoint is defined to accept POST requests.
   - It extracts the JSON data sent in the request body.
   - Validates that both `username` and `password` are present in the request.
   - If validation passes, it generates a JWT token and returns it in the response.

### Running the Flask Application:

You can run this application using the following command:

```sh
python your_flask_file.py
```

Replace `your_flask_file.py` with the actual filename where you saved the code. The app will run on `http://127.0.0.1:5000`.

### Example Request:

You can test this API using `curl`, Postman, or any HTTP client by sending a POST request to `/login` with a JSON body like this:

```sh
curl -X POST http://127.0.0.1:5000/login \
-H "Content-Type: application/json" \
-d '{"username": "user", "password": "pass"}'
```

This will return a JWT token.

### Note:
- This example uses `HS256` for the JWT signing algorithm, which is fine for testing but should be replaced with more secure methods like `RS256` or similar in production.
- Ensure your secret key (`SECRET_KEY`) is kept securely and not exposed in code repositories or deployed environments.
